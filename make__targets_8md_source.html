1 Makefile Targets {#makefile_targets}
2 ====================================
3 
4 While mavtables uses [CMake](https://cmake.org/) for building a simple Makefile
5 (which calls `cmake`) is included for convenience. The available targets are
6 listed below.
7 
8 
9 ## release (default)
10 
11 Build `mavtables` using the _Release_ option which generates an optimized (for
12 speed) executable without debug symbols. `mavtables` is placed at
13 `build/mavtables`.
14 
15 Unit tests are built as well under `build/unit_tests` and are built with
16 optimizations.
17 
18 
19 ## debug
20 
21 Build `mavtables` and unit tests using the _Debug_ option which will enable
22 debug flags and turn off optimizations. The executables are located at
23 `build/mavtables` and `build/unit_tests`.
24 
25 
26 ## test
27 
28 Build and run all tests. Runs both `test/unit_tests/run_tests.sh` and
29 `test/integration_tests/run_tests.sh`. The latter requires Python with the
30 packages located in `test/integration_tests/requirements.txt`.
31 
32 
33 ## unit_tests
34 
35 Build and run unit tests only.
36 
37 
38 ## integration_tests
39 
40 Build and run integration tests only. Requires Python with the packages located
41 in `test/integration_tests/requirements.txt`.
42 
43 
44 ## coverage
45 
46 Build and run all tests and compute test coverage. This option requires
47 [lcov](http://ltp.sourceforge.net/coverage/lcov.php) and
48 [gcovr](http://gcovr.com/) to be installed and will only work when the compiler
49 is `g++`.
50 
51 The coverage per file will be printed to the terminal and a detailed, line by
52 line, report generated with [lcov](http://ltp.sourceforge.net/coverage/lcov.php)
53 at `build/lcov/html/selected_targets/index.html`.
54 
55 
56 ## linecheck
57 
58 Prints all lines exceeding 80 characters.
59 
60 
61 ## style
62 
63 Fix the style of C++ source code and header files. The original files are
64 backed up with a `.orig` extension. [Artistic
65 Style](http://astyle.sourceforge.net/) is required to use this target. This
66 will also call the linecheck target.
67 
68 
69 ## html
70 
71 Generate html documentation with
72 [Doxygen](http://www.stack.nl/~dimitri/doxygen/) and place it at
73 `build/doc/html/index.html`.
74 
75 
76 ## doc
77 
78 Generate html and pdf documentation with
79 [Doxygen](http://www.stack.nl/~dimitri/doxygen/) and place it at
80 `build/doc/html/index.html` and `build/doc/html/mavtables.pdf` respectively.
81 
82 
83 ## gh-pages
84 
85 Generate html and pdf documentation and publish to
86 [https://shamuproject.github.io/mavtables](https://shamuproject.github.io/mavtables).
